# Quantum CoasterWorks v0.1.0-preview

`v0.1.0-preview` is the first public GitHub release for Quantum CoasterWorks.

## What This Release Is

This is an early backend/debug technical preview. It demonstrates the current engine-agnostic `Quantum.*` backend slice for centerline sampling, orientation/frame diagnostics, distance-based placeholder train data, debug snapshot contracts, validation, and thin local preview tools.

The release is meant for inspecting the backend architecture, contracts, test fixtures, generated debug output, and adapter boundary. It is also a public checkpoint for the current development direction before larger editor, renderer, and import surfaces are attempted.

## What This Release Is Not

This is not production-ready software, a polished coaster editor, a NoLimits replacement, a paid alpha, or a finished simulator.

It does not include a final frontend, final renderer, high-fidelity ride-through system, real train meshes, full NoLimits import, scenery import, terrain tools, scripting compatibility, production physics guarantees, or editor parity.

The SVG gallery and optional Blender viewer are debug/preview adapters only. They do not define the backend architecture and do not add frontend or renderer dependencies to core `Quantum.*` projects.

## Highlights

- Backend-first `Quantum.*` C# libraries remain engine-agnostic.
- `DebugViewportSnapshotV1` JSON contract for renderer-neutral debug data.
- Built-in debug viewport snapshot generation.
- CSV fixture-to-snapshot generation for self-authored synthetic fixture data.
- Snapshot validation and summary output.
- Multi-panel SVG debug previews with top-down and elevation/profile panels.
- Preview-only SVG smoothing for readability while preserving raw exported sample points.
- Static local SVG gallery generation under ignored `artifacts/debug-viewport/`.
- Optional Blender debug snapshot viewer under `tools/blender/`.
- Technical Preview demo script for repeatable local validation.

## How To Run Tests

From the repository root:

```powershell
dotnet test QuantumCoasterWorks.sln --nologo
```

## How To Run The Demo Script

On Windows, from the repository root:

```powershell
.\tools\demo-technical-preview-0.1.cmd
```

If direct script execution is blocked locally, the `.cmd` wrapper already runs PowerShell with a process-local execution policy bypass. The direct fallback is:

```powershell
powershell -ExecutionPolicy Bypass -File .\tools\demo-technical-preview-0.1.ps1
```

The demo runs the test suite, prints the `Quantum.Debug` help text, generates and validates built-in and CSV-derived `DebugViewportSnapshotV1` JSON snapshots, writes SVG previews, and builds a local HTML gallery.

Generated JSON, SVG, HTML, `.blend`, screenshots, render images, and release zip files are local artifacts. They should remain ignored unless intentionally attached to the GitHub Release later.

## How To Open The SVG Gallery

After running the demo script, open:

```text
artifacts/debug-viewport/index.html
```

The gallery is a static local HTML page that links to generated SVG previews and JSON snapshots. It is useful for quick inspection only; it is not a renderer or editor.

## Optional Blender Debug Viewer

The optional Blender adapter consumes generated `DebugViewportSnapshotV1` JSON snapshots from `artifacts/debug-viewport/`.

With Blender on `PATH`, run:

```powershell
blender --python tools/blender/import_debug_viewport_snapshot_v1.py -- artifacts/debug-viewport/DebugViewportSnapshotV1.sample.json
```

Or open Blender, load `tools/blender/import_debug_viewport_snapshot_v1.py` in the Scripting workspace, run the script, and choose a generated snapshot JSON file.

See `docs/visualization/blender-debug-viewer.md` for details.

## Known Limitations

- The CSV path is a sampled-frame debug/test bridge for self-authored or synthetic fixtures, not a full NoLimits project importer.
- Fixture coverage is intentionally small and synthetic.
- Debug viewport snapshots expose renderer-neutral centerline, frame, line, box, and optional train pose data only.
- Snapshot JSON does not define cameras, materials, meshes, editor UI, scene graph policy, or engine-specific coordinate conversion.
- SVG previews are technical debug aids. Their smoothing layer is visual-only and does not change JSON, spline, track, or physics behavior.
- Train visualization remains placeholder-oriented; simple boxes and diagnostic transforms are expected at this stage.
- Backend physics, force, and train systems are early foundations, not production simulation guarantees.
- Unity assets in this repository are optional/legacy debug visualization assets and should not drive backend architecture.

## Next Planned Milestones

- Strengthen stable centerline evaluation and station-distance semantics.
- Improve orientation frame stability and validation.
- Expand distance-based train car placement diagnostics.
- Add focused tests around frame stability, CSV fixture validation, train placement, and snapshot compatibility.
- Continue keeping debug viewers as thin adapters over backend JSON contracts.
- Keep editor, frontend, renderer, and engine choices exploratory until backend contracts are stronger.

## Release Checklist

Use this checklist after the release-prep changes are merged to `main`.

- [ ] Start from clean `main`.
- [ ] Run `dotnet test QuantumCoasterWorks.sln --nologo`.
- [ ] Run `.\tools\demo-technical-preview-0.1.cmd`.
- [ ] Optionally run a Blender import smoke test.
- [ ] Verify generated artifacts remain ignored and uncommitted.
- [ ] Create Git tag `v0.1.0-preview`.
- [ ] Create the GitHub Release from that tag.
- [ ] Optionally attach a locally generated demo gallery zip to the GitHub Release, but do not commit it.

Suggested command sequence:

```powershell
git checkout main
git pull --ff-only
git status --short

dotnet test QuantumCoasterWorks.sln --nologo
.\tools\demo-technical-preview-0.1.cmd

git status --short
git check-ignore -v artifacts/debug-viewport/DebugViewportSnapshotV1.sample.json artifacts/debug-viewport/index.html

git tag -a v0.1.0-preview -m "v0.1.0-preview"
git push origin v0.1.0-preview

gh release create v0.1.0-preview --title "Quantum CoasterWorks v0.1.0-preview" --notes-file docs/release/v0.1.0-preview.md --prerelease
```

Optional local gallery zip attachment:

```powershell
Compress-Archive -Path artifacts\debug-viewport\*.svg, artifacts\debug-viewport\index.html -DestinationPath artifacts\QuantumCoasterWorks-v0.1.0-preview-debug-gallery.zip -Force
gh release upload v0.1.0-preview artifacts\QuantumCoasterWorks-v0.1.0-preview-debug-gallery.zip --clobber
```
